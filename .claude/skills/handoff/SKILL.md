# handoff — 세션 인계

이 세션에서 한 일을 다음 세션이 **읽고 바로 이어갈 수 있도록** 단일 인계 파일에 정리한다.
산출물은 `장식이 아니라 정확성`이다. 추측·희망사항이 아니라 **검증된 사실과 막다른 길**을 적는다.

## 대상 파일

- 인자로 경로가 주어지면 그 파일에 쓴다. 없으면 프로젝트 루트의 **`.handoff.md`** (사용자가 `STATE`를 원하면 그쪽).
- 파일이 이미 있으면 **먼저 읽는다.** 끝난 항목은 ✅로 옮기고, 아직 유효한 미완료 항목은 ⏳에 carry-forward 한다. 통째로 덮어쓰지 말고 **현재 상태의 스냅샷으로 갱신**한다.

## 작성 절차

1. **컨텍스트 수집** — 다음을 실제로 실행해 근거를 확보한다(추측 금지):
   - `date '+%Y-%m-%d %H:%M'`, `git branch --show-current`
   - `git status --short`, `git diff --stat` (무엇이 실제로 바뀌었나)
   - 이번 세션에서 돌린 빌드/테스트/실행 명령의 **실제 출력**
2. **정직하게 분류** — 아래 세 구획을 채운다. 규칙:
   - ✅ 는 **검증된 것만.** 증거(`명령` · `파일:라인` · 테스트 결과)를 같은 줄에 붙인다. 증거 없으면 ✅ 아님 → ⏳로.
   - 테스트 실패·스킵·미검증이면 그대로 적는다. "되겠지"로 ✅ 표시 금지.
   - ❌ 는 **시도했지만 안 된 접근**과 **원인**을 남겨 다음 세션의 반복을 막는다.
3. **파일 쓰기** — 아래 템플릿으로 갱신하고, 경로를 사용자에게 알린다.

## 출력 템플릿

```markdown
# 🤝 Handoff — <한 줄 현재 상태>

> 갱신: <YYYY-MM-DD HH:MM> · 브랜치: `<branch>` · 변경: `<git status --short 요약(신규/untracked 포함) — tracked 변경은 git diff --stat>`

## ✅ 완료 (검증된 것만)
- [x] <한 일> — 증거: `<실행한 명령 / 파일:라인 / 테스트 결과>`

## ❌ 시도 → 실패 / 막힘 (반복 금지)
- [ ] <시도한 접근> — 원인: <왜 안 됐나> → 다음 대안: <있으면>

## ⏳ 다음 할 일 / 미착수
- [ ] <바로 이어갈 다음 단계> — 시작점: `<파일 / 명령>`
- [ ] <아직 손 안 댄 것>

## 📌 컨텍스트 메모 (선택)
- 결정/제약/주의: <이어받는 사람이 모르면 헤맬 것들 — 환경 설정, 합의된 방향, 함정>
```

## 원칙

- **간결·스캔 가능하게.** 각 항목 한 줄. 산문 단락 금지.
- **다음 세션 관점에서 써라.** "내가 뭘 했나"가 아니라 "다음 사람이 뭘 알아야 이어가나".
- ✅/❌/⏳ 세 구획은 비어도 헤더는 남긴다(비었으면 `- 없음`).
- 이 저장소 규칙([.claude/rules/anti-ai-slop.md](../../rules/anti-ai-slop.md))의 정신대로 이모지 헤더 외 장식은 넣지 않는다.
